<Note>
  Tasks represent asynchronous operations that handle content processing, feature extraction, and index management. They provide real-time status tracking and error handling capabilities.
</Note>

## Key Concepts

<CardGroup cols={2}>
  <Card title="Asynchronous Processing" icon="clock">
    Non-blocking operations that:
    - Run in the background
    - Support parallel execution
    - Handle long-running processes
  </Card>
  
  <Card title="Status Tracking" icon="chart-line">
    Monitor task lifecycle with:
    - Real-time status updates
    - Progress indicators
    - Completion notifications
  </Card>
</CardGroup>

## Task Lifecycle

<Steps>
  <Step title="Initialization">
    Task is created and queued for processing
  </Step>
  <Step title="Processing">
    Content is downloaded and processed according to task type
  </Step>
  <Step title="Completion">
    Task finishes with success or failure status
  </Step>
</Steps>

## Task States

<AccordionGroup>
  <Accordion title="Processing States" icon="spinner">
    - `INITIALIZING` - Task setup and validation
    - `QUEUED` - Waiting for processing
    - `DOWNLOADING` - Retrieving content
    - `PROCESSING` - Active execution
    - `UPLOADING` - Saving results
  </Accordion>

  <Accordion title="Final States" icon="flag-checkered">
    - `DONE` - Successfully completed
    - `FAILED` - Error occurred
    - `SKIPPED` - Task ignored
    - `CANCELLED` - Manually terminated
  </Accordion>
</AccordionGroup>

## Example Usage

```python
# Check task status
task_status = mixpeek.tasks.get(task_id="task_123")

if task_status.status == "DONE":
    results = task_status.outputs
    # Process results
elif task_status.status == "FAILED":
    error = task_status.error
    # Handle error
```
