---
title: "Full File"
description: "Retrieve all the contents of a file by its file_id."
api: "GET /collections/file/<file_id>/full"
---

Once a file is indexed, Mixpeek has automatically pulled out data including `embeddings`, `transcriptions`, `labels`, `descriptions`, and more. By calling this API, you can retrieve all the indexed data associated with the `file_id`.

## Request

<ParamField url="file_id" type="string" required>
  The ID of the file to retrieve the full contents for
</ParamField>

## Response

<ResponseField name="index_id" type="string">
  The unique identifier for the indexed file
</ResponseField>

<ResponseField name="file_id" type="string">
  The unique identifier for the file
</ResponseField>

<ResponseField name="collection_id" type="string">
  The ID of the collection the file belongs to
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the file processing
</ResponseField>

<ResponseField name="url" type="string">
  The URL of the original file
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional metadata about the file and its context
</ResponseField>

<ResponseField name="created_at" type="string">
  The timestamp when the file was created or indexed
</ResponseField>

<ResponseField name="video_segments" type="object">
  Contains grouped segments of the video content
</ResponseField>
