---
title: "Text"
description: "Search indexed content using text queries with customizable filters and pagination."
api: "POST /search/text"
---

The `Search by Text` endpoint allows you to search for content using a text query. This is useful for finding relevant information across your indexed content.

## Request

<ParamField body="input" type="string" required>
  The text query to search for
</ParamField>

<ParamField body="modality" type="string" required>
  The modality to search in (e.g., "text", "video", "image")
</ParamField>

<ParamField body="input_type" type="string" required>
  The type of input (e.g., "text")
</ParamField>

<ParamField body="filters" type="object" optional>
  Additional filters to apply to the search
</ParamField>

<ParamField body="group_by_file" type="boolean" optional>
  Whether to group results by file
</ParamField>

<ParamField body="page" type="integer" optional>
  The page number for pagination
</ParamField>

<ParamField body="page_size" type="integer" optional>
  The number of results per page
</ParamField>

<RequestExample>

```python Python
from mixpeek import Mixpeek

client = Mixpeek('<API_KEY>')

response = client.search.text(
    input="<str>",
    modality="<str>",
    input_type="<str>",
    filters={"$or": [{"collection_id": "<str>"}]},
    group_by_file=<bool>,
    page=<int>,
    page_size=<int>
)
```

```shell Shell
curl --location 'https://api.mixpeek.com/search/text' \
--header 'Authorization: Bearer <API_KEY>' \
--header 'Content-Type: application/json' \
--data '{
    "modality": "<str>",
    "input": "<str>",
    "group_by_file": <bool>,
    "filters": {
        "$or": [
            {
                "collection_id": "<str>"
            }
        ]
    }
}'
```

</RequestExample>
