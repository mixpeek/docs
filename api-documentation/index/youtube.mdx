---
title: "YouTube"
description: "Index content from YouTube videos with optional metadata and processing settings."
api: "POST /index/youtube"
---

The `Index by YouTube` endpoint allows you to index content from a specified YouTube video. This is useful for processing and indexing video content directly from YouTube.

## Request

<ParamField body="youtube_video_id" type="string" required>
  The ID of the YouTube video to be indexed
</ParamField>

<ParamField body="collection_id" type="string" required>
  The ID of the collection to index the content into
</ParamField>

<ParamField body="metadata" type="object" optional>
  Additional metadata to associate with the indexed content
</ParamField>

<ParamField body="settings" type="object" optional>
  Optional settings for processing the content
</ParamField>

## Response

<ResponseField name="message" type="string">
  A message indicating the status of the indexing process
</ResponseField>

<ResponseField name="task_id" type="string">
  A unique identifier for the indexing task
</ResponseField>

<ResponseExample>

```json Response
{
  "message": "YouTube video processing started",
  "task_id": "<str>"
}
```

</ResponseExample>

The indexing process is asynchronous. You can use the returned `task_id` to check the status of the indexing task if needed.
