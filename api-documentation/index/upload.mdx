---
title: "Upload"
description: "Upload and index files directly with optional metadata and processing settings."
api: "POST /index/upload"
---

The `Index by File Upload` endpoint allows you to upload and index a file directly. This is useful for processing and indexing local files or documents.

## Request

<ParamField body="file" type="file" required>
  The file to be uploaded and indexed
</ParamField>

<ParamField body="collection_id" type="string" required>
  The ID of the collection to index the content into
</ParamField>

<ParamField body="metadata" type="object" optional>
  Additional metadata to associate with the indexed content
</ParamField>

<ParamField body="settings" type="object" optional>
  Optional settings for processing the content (e.g., image caption model)
</ParamField>

<RequestExample>

```shell Shell
curl --location 'https://api.mixpeek.com/index/upload' \
--header 'Authorization: Bearer API_KEY' \
--form 'file=@"/path/to/your/file.jpg"' \
--form 'collection_id="testing"' \
--form 'metadata="{\"foo\":\"bar\"}"'
```

</RequestExample>

## Response

<ResponseField name="message" type="string">
  A message indicating the status of the indexing process
</ResponseField>

<ResponseField name="task_id" type="string">
  A unique identifier for the indexing task
</ResponseField>

<ResponseExample>

```json Response
{
  "message": "File upload processing started",
  "task_id": "<str>"
}
```

</ResponseExample>

The indexing process is asynchronous. You can use the returned `task_id` to check the status of the indexing task if needed.
