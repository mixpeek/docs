<Note>

</Note>

## Health checks

- **API**: `/v1/health` returns status for Redis, MongoDB, Qdrant, Celery, and Engine
- **Reference**: [Health API](/api-reference/health/healthcheck)

```bash
curl -s https://api.mixpeek.com/v1/health | jq
```

## Metrics

<Tabs>
  <Tab title="API">
    - Request throughput, latency percentiles, error rates
    - Rate limit hits per route and namespace
  </Tab>
  
  <Tab title="Engine">
    - Task queue depth, execution time, success/failure counts
    - Extractor runtimes, clustering metrics, GPU/CPU utilization
  </Tab>
  
  <Tab title="Datastores">
    - Qdrant memory/IO, vector count, index build time
    - MongoDB ops/latency, Redis connections, S3 errors
  </Tab>
</Tabs>

## Tasks visibility

- Use [Tasks](/processing/tasks) to monitor batch and long‑running jobs
- List active tasks and drill into details; alert on FAILED/CANCELED
