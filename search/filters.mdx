<Note>
  Search filters allow you to refine results using logical operators and field conditions, helping you find exactly what you need.
</Note>

## Filter Types

<CardGroup cols={2}>
  <Card title="Logical Operators" icon="code-branch">
    - AND: All conditions must match
    - OR: Any condition can match
    - NOR: No conditions should match
    - Nested combinations supported
  </Card>
  
  <Card title="Field Operators" icon="equals">
    - Comparison (eq, neq, gt, lt)
    - Range (gte, lte)
    - Collection (in, nin)
    - Existence & Pattern (exists, regex)
  </Card>
</CardGroup>

## Filter Structure

<CodeGroup>
  ```json Basic Filter
  {
    "filters": {
      "AND": [
        {
          "key": "category",
          "operator": "eq",
          "value": "electronics"
        }
      ]
    }
  }
  ```

  ```json Complex Filter
  {
    "filters": {
      "case_sensitive": true,
      "AND": [
        {
          "key": "price",
          "operator": "gte",
          "value": 100
        },
        {
          "key": "in_stock",
          "operator": "eq",
          "value": true
        }
      ],
      "OR": [
        {
          "key": "category",
          "operator": "in",
          "value": ["electronics", "accessories"]
        }
      ]
    }
  }
  ```
</CodeGroup>

## Supported Operators

<AccordionGroup>
  <Accordion title="Comparison Operators">
    | Operator | Description | Example |
    |----------|-------------|---------|
    | `eq` | Exact match | `"operator": "eq"` |
    | `neq` | Not equal | `"operator": "neq"` |
    | `gt` | Greater than | `"operator": "gt"` |
    | `gte` | Greater than or equal | `"operator": "gte"` |
    | `lt` | Less than | `"operator": "lt"` |
    | `lte` | Less than or equal | `"operator": "lte"` |
  </Accordion>

  <Accordion title="Collection Operators">
    | Operator | Description | Example |
    |----------|-------------|---------|
    | `in` | Value in array | `"operator": "in"` |
    | `nin` | Value not in array | `"operator": "nin"` |
    | `exists` | Field exists | `"operator": "exists"` |
    | `regex` | Regular expression match | `"operator": "regex"` |
  </Accordion>
</AccordionGroup>

## Common Use Cases

<Tabs>
  <Tab title="Metadata Filtering">
    ```json
    {
      "filters": {
        "AND": [
          {
            "key": "author",
            "operator": "eq",
            "value": "John Doe"
          },
          {
            "key": "date",
            "operator": "gte",
            "value": "2024-01-01"
          }
        ]
      }
    }
    ```
  </Tab>

  <Tab title="Content Filtering">
    ```json
    {
      "filters": {
        "AND": [
          {
            "key": "content_type",
            "operator": "in",
            "value": ["article", "blog"]
          },
          {
            "key": "language",
            "operator": "eq",
            "value": "en"
          }
        ]
      }
    }
    ```
  </Tab>
</Tabs>

## Best Practices

<Steps>
  <Step title="Use Indexed Fields">
    Filter on indexed fields for optimal performance
  </Step>
  <Step title="Optimize Conditions">
    Order conditions from most to least selective
  </Step>
  <Step title="Combine Related Filters">
    Group related conditions under single operators
  </Step>
  <Step title="Validate Data Types">
    Ensure correct data types for each operator
  </Step>
</Steps>

## Limitations

<Warning>
  Be aware of these technical constraints when building filters:
  - Maximum 20 conditions per operator
  - Only indexed fields can be filtered
  - Nested filters have performance impact
  - Case-sensitive filtering requires more resources
</Warning>

For detailed implementation examples, see the [Search API Reference](/api-reference/search/overview).
