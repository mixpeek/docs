---
title: "Embed"
description: "Generates embeddings for text, image, audio, or video data to create machine-readable representations."
api: "POST /embed/{modality}"
---

The embed method allows you to generate embeddings from various modalities. Depending on the modality, different techniques can be applied:

- **image**: feature extraction, dimensionality reduction, etc.
- **audio**: spectral analysis, feature extraction, etc.
- **video**: frame analysis, feature extraction, etc.
- **text**: vectorization, semantic analysis, etc.

## Request

<ParamField body="model" type="string" required>
  The model to use for embedding
</ParamField>

<ParamField body="input" type="string" required>
  The input source for the file to be embedded, e.g., a URL or direct content.
</ParamField>

<ParamField body="modality" type="string" required>
  The type of data to be embedded (e.g., text, image, audio, video).
</ParamField>

<ParamField body="input_type" type="string" required>
  The type of input provided (e.g., "url" for remote URLs, "content" for direct
  base64-encoded content).
</ParamField>

<RequestExample>

```python Python
from mixpeek import Mixpeek

mixpeek = Mixpeek("API_KEY")
embedding = mixpeek.embed(
  modality="video",
  model="mixpeek/vuse-generic-v1",
  input="s3://waving_boy.mp4",
  input_type="url"
)
```

```shell Shell
curl --location 'https://api.mixpeek.com/embed/{modality}' \
--header 'Authorization: Bearer API_KEY' \
--header 'Content-Type: application/json' \
--data '{
    "input": "s3://waving_boy.mp4",
    "modality": "video",
    "model": "mixpeek/vuse-generic-v1",
    "input_type": "url"
}'
```
