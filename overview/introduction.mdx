<Note>
  Welcome to the Mixpeek documentation! Use our HTTP API to ingest multimodal data, extract features, enrich documents, and retrieve results with hybrid search.
</Note>

> Mixpeek turns raw media (text, images, video, audio, PDFs) into structured **documents** and optimized **feature stores** you can query with flexible, staged **retrievers**. The API provides clear primitives for isolation, ingestion, enrichment, and retrieval.

## How It Works

<Steps>
  <Step title="Create a Namespace (optional)">
    Use [**Namespaces**](/ingestion/namespaces) to establish isolation boundaries for data and queries. Pass `X-Namespace` on subsequent calls.
  </Step>
  <Step title="Upload Objects to Buckets">
    Create a [**Bucket**](/ingestion/buckets) and add [**Objects**](/ingestion/objects) single or batch. Group work with [**Batches**](/processing/batching) and submit for processing.
  </Step>
  <Step title="Define Collections & Feature Extractors">
    Create a [**Collection**](/ingestion/collections) with input/output schemas and [**Feature Extractors**](/processing/feature-extractors). Describe available features to understand addresses and indexes.
  </Step>
  <Step title="Enrich with Taxonomies & Clustering">
    Configure [**Taxonomies**](/enrichment/taxonomies) and [**Clusters**](/enrichment/clusters) to enrich and organize documents.
  </Step>
  <Step title="Retrieve & Analyze">
    Use [**Retrievers**](/retrieval/retrievers) to combine vector search, filters, grouping, and ranking, optionally generating presigned URLs.
  </Step>
</Steps>

## Headers You’ll Use

```bash
Authorization: Bearer sk_your_api_key
X-Namespace: your_namespace_or_id  # optional but recommended for isolation
```

## Common Use Cases

<Tabs>
  <Tab title="Multimodal Search">
    - Index images, PDFs, video, and text into **Collections** with appropriate **feature_extractors**
    - Execute a **Retriever** that combines vector similarity, metadata filters, and reranking
    - Optional: return presigned URLs for assets in results (`return_urls=true`)
  </Tab>
  
  <Tab title="Content Analytics">
    - Use **Batches** for large-scale ingestion and processing
    - Apply **Clustering** for grouping, and **Taxonomies** for structured tagging
    - Stream cluster artifacts or members as needed in [Clusters](/enrichment/clusters)
  </Tab>
  
  <Tab title="Training Data">
    - Organize paired multimodal examples as **Objects**
    - Persist derived annotations as **Documents** with lineage to source objects
    - Retrieve subsets with filters and grouping for training pipelines
  </Tab>
  
  <Tab title="Operational Workflows">
    - Track long-running jobs via Tasks and integrate Webhooks for eventing
    - Manage API keys and permissions under **Organizations**
    - Partition environments by **Namespaces** (dev/staging/prod)
  </Tab>
</Tabs>

<Note>
  Mixpeek’s API gives you typed resources—Namespaces, Buckets/Objects/Batches, Collections/Documents/Feature Extractors, Taxonomies, Clusters, Retrievers, Tasks, and Webhooks—to build reliable multimodal applications end‑to‑end.
</Note>
