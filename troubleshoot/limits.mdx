<Note>
  Requests are governed by account tier, per‑route permissions, and Redis‑backed rate limits. This guide explains tiers, defaults, and how to resolve 429 Too Many Requests.
</Note>

## Permissions

- API keys carry permissions such as <code>view</code>, <code>create</code>, <code>update</code>, <code>delete</code>
- Admin levels: <code>admin</code>, <code>super</code>
- Data scopes: <code>read_public</code>, <code>read_private</code>, <code>write_public</code>, <code>write_private</code>

If a route requires permissions you lack, calls may return 403 Unauthorized.

## 429 Too Many Requests

<Steps>
  <Step title="Identify the route">
    The <code>details.api_name</code> field indicates which bucket exceeded limits
  </Step>
  <Step title="Back off">
    Apply exponential backoff (e.g., 1s, 2s, 4s, jitter)
  </Step>
  <Step title="Reduce concurrency">
    Lower parallel calls to the route; batch where possible
  </Step>
  <Step title="Upgrade tier">
    Consider Pro/Enterprise for higher limits if usage is sustained
  </Step>
</Steps>

## Best practices
